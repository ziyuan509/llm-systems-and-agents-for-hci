# Related Work Map

## Cluster 1: Agent Transparency, Debugging, and Runtime Steering

Use this cluster to position against systems that emphasize traces, states, and developer-facing intervention.

Key question:

- Why is delegation governance a different interaction problem from trace visibility?

## Cluster 2: End-User Control of Generative Systems

Use this cluster to connect with prompt editing, workflow customization, and mixed-initiative control.

Key question:

- Why are policy objects a better control surface than raw prompt edits for ordinary users?

## Cluster 3: Privacy, Safety, and Policy Enforcement in Agents

Use this cluster to motivate why governable surfaces include memory, tools, disclosure, and escalation.

Key question:

- Which backend controls should become user-visible interaction objects?

## Cluster 4: Longitudinal Personal Agent Governance

Use this cluster to frame ongoing revision, accountability, and sustained agent use over time.

Key question:

- How should delegation contracts evolve across tasks instead of only within a session?
